## Quiz

### SLO 1: Problem-solving and pseudocode quality

#### Question 1 (SLO 1)

You need a class that stores **a single value of any type** and returns it on demand. The same source code should support `Box<Integer>`, `Box<String>`, `Box<Cat>`, and so on, with the compiler enforcing the choice at every call site. Which declaration is **best**?

* **A)**
    ```java
    public class Box {
        private Object item;
        public void setItem(Object item) { this.item = item; }
        public Object getItem()          { return item; }
    }
    ```
* **B)**
    ```java
    public class Box<T> {
        private T item;
        public void setItem(T item) { this.item = item; }
        public T getItem()          { return item; }
    }
    ```
* **C)**
    ```java
    public class Box<T extends Box<T>> { ... }
    ```
* **D)**
    ```java
    public class Box<Integer> { private Integer item; ... }
    ```
* **E)**
    ```java
    public class Box implements Generic { ... }
    ```

**Your answer:** _B_

#### Question 2 (SLO 1)

You are writing a `static` method `max` that returns the larger of two values. The values can be `Integer`, `String`, or any other type that has a `compareTo` method. Which signature is **best**?

* **A)** `public static Object max(Object a, Object b)`
* **B)** `public static <T> T max(T a, T b)` (no bound)
* **C)** `public static <T extends Comparable<T>> T max(T a, T b)`
* **D)** `public static T max(T a, T b)` (no `<T>` declaration)
* **E)** `public static <T> Object max(Object a, Object b)`

**Your answer:** _C_

#### Question 3 (SLO 1)

A program needs to keep a list of mixed `Animal` subclasses (a `Dog` here, a `Cat` there, a `Bird` over there) and pass them around uniformly. Which declaration is the **best** fit?

* **A)** `ArrayList list = new ArrayList();`
* **B)** `ArrayList<Object> list = new ArrayList<>();`
* **C)** `ArrayList<Animal> list = new ArrayList<>();`
* **D)** `ArrayList<Dog> list = new ArrayList<>();`
* **E)** `Animal[] list = new Animal[10];`

**Your answer:** _C_

#### Question 4 (SLO 1)

You need a generic class `Pair` that holds two values of **two different types** -- for example, a `Pair<String, Integer>` for a name and an age, or a `Pair<Cat, Date>` for a cat and the date it was adopted. Which declaration is **best**?

* **A)**
    ```java
    public class Pair<T> {
        private T first;
        private T second;
    }
    ```
* **B)**
    ```java
    public class Pair<K, V> {
        private K first;
        private V second;
    }
    ```
* **C)**
    ```java
    public class Pair<First, Second> implements Pair<First, Second> { ... }
    ```
* **D)**
    ```java
    public class Pair {
        private Object first;
        private Object second;
    }
    ```
* **E)**
    ```java
    public class Pair<T extends Comparable<T>> {
        private T first;
        private T second;
    }
    ```

**Your answer:** _B_

### SLO 2: Code correctness, efficiency, and implementation

#### Question 5 (SLO 2)

What happens when this program is compiled and run?

```java
import java.util.ArrayList;

public class Demo {
    public static void main(String[] args) {
        ArrayList list = new ArrayList();
        list.add("hello");
        list.add(42);
        String s = list.get(0);
        System.out.println(s);
    }
}
```

* **A)** It compiles and runs; the program prints `hello`.
* **B)** The program **does not compile**: `list.get(0)` returns `Object` from the raw `ArrayList`, which cannot be assigned to a `String` without an explicit cast.
* **C)** It compiles but throws a `ClassCastException` at run time.
* **D)** It compiles and runs; the integer `42` is automatically converted to the string `"42"`.
* **E)** It compiles but the list ends up empty.

**Your answer:** _B_

#### Question 6 (SLO 2)

What does this program print?

```java
class Box<T> {
    private T item;
    public void setItem(T item) { this.item = item; }
    public T getItem()          { return item; }
}

public class Demo {
    public static void main(String[] args) {
        Box<Integer> a = new Box<>();
        a.setItem(7);
        Box<String> b = new Box<>();
        b.setItem("hi");
        System.out.println(a.getItem() + " " + b.getItem());
    }
}
```

* **A)** `7 hi`
* **B)** `null null`
* **C)** Compile error: `T` is not a real type.
* **D)** A runtime exception is thrown.
* **E)** `Box{7} Box{hi}`

**Your answer:** _A_

* **A)** line A

#### Question 8 (SLO 2)

```java
public static <T extends Comparable<T>> T max(T a, T b) {
```

```java
System.out.println(max(3, 5));
System.out.println(max("apple", "banana"));
```

* **A)**

* **B)**
    ```
    5
    apple
    ```
* **C)**
    ```
    3
    banana
    ```

    ```
    apple
    ```

### SLO 3: Documentation, testing, and best-practice judgement

#### Question 9 (SLO 3)

Which is the **best** Javadoc for a generic class declared as `public class Box<T>` with the methods `setItem(T)` and `getItem()`?

* **A)** `// Box`
* **B)**
    ```java
    /** A box. */
    ```
* **C)**
    ```java
    /**
     * Generic single-element container.
     *
     * @param <T> the type of element this box holds.
     */
    public class Box<T> { ... }
    ```
* **D)** No comment at all (the class name is self-explanatory).
* **E)**
    ```java
    /** @return Box */
    public class Box<T> { ... }
    ```

**Your answer:** _C_

#### Question 10 (SLO 3)

When is a **bounded** type parameter `<T extends X>` the right choice rather than an **unbounded** `<T>`?

* **A)** Always: bounds make the program faster.
* **B)** Never: bounds slow down compilation and add no value.
* **C)** When the **body** of the class or method needs to call methods declared on `X`. The bound is the compile-time guarantee that every `T` will have those methods.
* **D)** Only when `X` is a class (not an interface).
* **E)** Only when `T` is meant to hold a primitive type.

**Your answer:** _C_

## Quiz

### Question 1

Which of the following statements about the `main` method signature `public static void main(String[] args)` is **incorrect**?

* **A)** `public` allows the JVM to invoke the method from outside the class.
* **B)** `static` lets the method run without first creating an object of the class.
* **C)** `void` means the method does not return a value.
* **D)** `String[] args` receives command-line arguments as an array of strings.
* **E)** `main` is just a conventional name and may be renamed (for example, to `start`) as long as the file extension stays `.java`.

**Your answer:** _E_

### Question 2

A student writes a file called `Greeting.java` containing the line `public class greeting {` and tries to compile with `javac Greeting.java`. What happens?

* **A)** The file compiles and runs normally; Java is case insensitive about class names.
* **B)** The file compiles but fails at run time with a `NoSuchMethodError`.
* **C)** Compilation fails with an error stating that the public class should be declared in a file named `greeting.java`.
* **D)** The file compiles successfully and produces a class file called `Greeting.class`.
* **E)** Compilation produces a warning but continues normally; the resulting program prints nothing.

**Your answer:** _C_

### Question 3

Which of the following statements about Java types is **incorrect**?

* **A)** The literal `3.14` is a `double` by default; to write a `float` literal you must add an `f` suffix, as in `3.14f`.
* **B)** `String` is a class, not a primitive type.
* **C)** `'A'` is a `char` (a 16-bit Unicode character) and `"A"` is a `String`.
* **D)** Integer overflow on an `int` throws an `ArithmeticException` at run time.
* **E)** A `char` literal is enclosed in single quotes; a `String` literal is enclosed in double quotes.

**Your answer:** _D_

### Question 4

What does the following program print?

```java
public class Demo {
    public static void main(String[] args) {
        int a = 5;
        int b = 2;
        double result = a / b;
        System.out.println(result);
    }
}
```

* **A)** `2.5`
* **B)** `2.0`
* **C)** `3.0`
* **D)** `3`
* **E)** `2`

**Your answer:** _B_

### Question 5

* **A)** `3.4 0`

* **D)** `3 0`
* **E)** `2 3`

### Question 6

Consider the following statements about Java's logical operators:

1. `&&` short-circuits: if the left operand is `false`, the right operand is not evaluated.
2. `||` short-circuits: if the left operand is `true`, the right operand is not evaluated.
3. The expression `s != null && s.length() > 0` is safe to evaluate even when `s` is `null`.
4. The expression `s.length() > 0 && s != null` is safe to evaluate even when `s` is `null`.
5. `!` (NOT) has higher precedence than `&&`, which has higher precedence than `||`.

Which option lists **all** of the correct statements?

* **A)** 1, 2, 3, 5
* **B)** 1, 2, 4, 5
* **C)** 1, 3, 5
* **D)** 2, 4, 5
* **E)** 1, 2, 3, 4, 5

**Your answer:** _A_

### Question 7

Which of the following statements about pitfalls in Java's `if` statements is described **incorrectly**?

* **A)** Writing `if (x > 0);` ends the `if` immediately, so the block that follows runs unconditionally.
* **B)** Inside the condition of an `if`, you should use `==` to test, not `=` to assign.
* **C)** Always use curly braces around the body, even when the body is a single line.
* **D)** Comparing two `String` values for **content** equality should use `==`.
* **E)** Putting the cheap or null-checking condition on the left side of `&&` lets short-circuit evaluation protect the right side.

**Your answer:** _D_

### Question 8

What does the following program print?

```java
public class Sign {
    public static void main(String[] args) {
        int x = 0;
        if (x < 0) {
            System.out.println("negative");
        } else if (x == 0) {
            System.out.println("zero");
        } else {
            System.out.println("positive");
        }
        System.out.println("done");
    }
}
```

* **A)** `negative`
* **B)** `zero`
* **C)** `positive`
* **D)** `zero` followed by `done` on the next line
* **E)** `positive` followed by `done` on the next line

**Your answer:** _D_

### Question 9

Which of the following best captures the relationship between a **class** and an **object** in Java?

* **A)** A class is a runtime instance of an object.
* **B)** A class is a blueprint; an object is a runtime instance of the class, created with the `new` keyword.
* **C)** Class and object are interchangeable terms in Java.
* **D)** An object is a category and a class is one of its members.
* **E)** Objects are stored in `.java` files and classes are stored in `.class` files.

**Your answer:** _B_

### Question 10

Consider these statements about classes and access in Java:

1. A `public` field can be read and written from anywhere.
2. A `private` field can be read and written only from inside the same class.
3. A constructor has the same name as the class and no return type.
4. A mutator (setter) returns the value of a private field.
5. An accessor (getter) returns the value of a private field.

Which option lists **all** of the correct statements?

* **A)** 1, 2, 3, 4
* **B)** 1, 2, 3, 5
* **C)** 1, 3, 5
* **D)** 2, 4, 5
* **E)** 1, 2, 3, 4, 5

**Your answer:** _A_

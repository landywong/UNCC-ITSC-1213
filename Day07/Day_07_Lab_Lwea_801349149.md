### Question 1

The `Player` class declares its three fields as `private`:

```java
private String name;
private int    hp;
private int    attackPower;
```

A separate test class writes:

```java
Player player = new Player("Hero", 30, 10);
player.attackPower = 999;
```

What happens when you compile this test class?

* **A)** It compiles and runs; `attackPower` becomes 999.
* **B)** It compiles, but at runtime the assignment is silently ignored.
* **C)** It does **not** compile; `attackPower` has private access in `Player`.
* **D)** It compiles and runs, but the JVM throws an `IllegalAccessException`.
* **E)** It compiles and changes the `name` field to 999 instead.

**Your answer:** _C_

### Question 2

The `Player` constructor begins with:

```java
public Player(String name, int hp, int attackPower) {
    this.name = name;
    this.hp = hp;
    this.attackPower = attackPower;
}
```

What would happen if every `this.` were removed (so the body became `name = name;`, `hp = hp;`, and `attackPower = attackPower;`)?

* **A)** The compiler refuses the file because of the duplicate names.
* **B)** The class compiles, but each line assigns the parameter to itself; the fields remain at their default values (`null`, `0`, `0`).
* **C)** The class compiles, and Java automatically assigns the parameter to the matching field even without `this`.
* **D)** The constructor still works because Java throws an exception that is silently caught.
* **E)** The compiler renames the parameters to avoid the conflict.

**Your answer:** _B_

### Question 7

### Question 10

`BattleSimulation` calls `displayStatus(players, monsters)` at the end of every round. The `players` array is created once in `main`, and only the **state of each `Player` object inside it** changes (HP goes down). Which statement best describes the relationship between the `players` parameter inside `displayStatus` and the `players` array inside `main`?

* **A)** `displayStatus` receives a copy of the array, so changes made by `monster.attack` cannot be seen there.
* **B)** Both names refer to the same array of `Player` objects in memory; any change to a `Player`'s HP made through one is visible through the other.
* **C)** The two arrays are completely independent; Java duplicates every object on every method call.
* **D)** `players` is a global variable, so it is shared automatically and the parameter is unused.
* **E)** The parameter is `null` until you assign it inside the method body.

**Your answer:** _B_

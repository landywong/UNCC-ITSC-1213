## Quiz

### Question 1

Open `Transaction.java`. How is `Transaction` declared at the top of the file?

* **A)** `public abstract class Transaction`
* **B)** `public interface Transaction`
* **C)** `public class Transaction implements Banking`
* **D)** `public enum Transaction`
* **E)** `public record Transaction`

**Your answer:** _B_

### Question 2

In `Transaction.java`, what is the **return type** of the method declared as `void deposit(double amount);`?

* **A)** `void`
* **B)** `double`
* **C)** `boolean`
* **D)** `int`
* **E)** `Transaction`

**Your answer:** _A_

### Question 3

Inside `BankDemo.main`, the body of `try { ... }` is followed by a sequence of `catch` blocks. Which exception type is caught **first** (in the topmost `catch` block)?

* **A)** `Exception`
* **B)** `NullPointerException`
* **C)** `IllegalArgumentException`
* **D)** `IOException`
* **E)** `ArithmeticException`

**Your answer:** _C_

### Question 4

Look at the `finally` clause at the bottom of `BankDemo.main`. What does that `finally` block actually do?

* **A)** Prints `"Goodbye"` and exits the program.
* **B)** Throws a fresh exception so the user knows the program ended.
* **C)** Calls `scanner.close();` to release the `Scanner` resource.
* **D)** Re-prompts the user to retype the failed input.
* **E)** Resets the account's balance to zero.

**Your answer:** _C_

### Question 5

Open `BankAccountTest.java`. The first test, `testImplementsTransaction`, contains the expression `account instanceof Transaction`. What is this test actually verifying?

* **A)** That `Transaction` is itself an interface.
* **B)** That the `BankAccount` class implements the `Transaction` interface (so a `BankAccount` instance is-a `Transaction`).
* **C)** That the constructor accepts a negative initial balance.
* **D)** That `getBalance()` always returns the literal value `100.0`.
* **E)** That the call `deposit(0)` runs without throwing anything.

**Your answer:** _B_

### Question 6

The test method `testConstructorNegativeBalance` calls `new BankAccount(-50.0)` inside a `try`. What does the test treat as a **PASS**?

* **A)** The constructor returns `null` instead of an object.
* **B)** The constructor silently sets the balance to `0`.
* **C)** The constructor throws an `IllegalArgumentException`.
* **D)** The constructor throws an `ArithmeticException`.
* **E)** Nothing throws, and the test always prints `PASS`.

**Your answer:** _C_

### Question 7

In `testWithdrawInsufficientFunds`, an account is created with a balance of `100.0` and then `withdraw(200.0)` is called. According to the test, what is the **correct** behaviour of `withdraw` for this case?

* **A)** The balance silently becomes `-100.0`.
* **B)** `withdraw` returns `false` and the balance is unchanged.
* **C)** `withdraw` prints an error message and continues.
* **D)** `withdraw` throws an `IllegalArgumentException`.
* **E)** `withdraw` throws a `NullPointerException`.

**Your answer:** _D_

### Question 8

The test method `testDepositNegative` calls `account.deposit(-20.0)` inside a `try` and expects a specific exception type in the matching `catch`. Which exception type is it?

* **A)** `IllegalArgumentException`
* **B)** `IOException`
* **C)** `NullPointerException`
* **D)** `ArithmeticException`
* **E)** `RuntimeException` (the bare base class)

**Your answer:** _A_

### Question 9

Count the **distinct test methods** that are declared as `public static void test...()` in `BankAccountTest.java` and that are called from `main`. How many are there?

* **A)** 4
* **B)** 6
* **C)** 7
* **D)** 8
* **E)** 10

**Your answer:** _D_

### Question 10

Putting the three files together, which of the following is the **most accurate** description of the `BankAccount` class that a student would need to write so that `BankAccountTest` reports `PASS` on every test and `BankDemo` runs as written?

* **A)** A `public class BankAccount` with public fields and no constructor.
* **B)** A `public class BankAccount extends Transaction` with a single `double balance` field and `deposit`/`withdraw` methods that silently ignore negative inputs.
* **C)** A `public class BankAccount implements Transaction` that has a private `double balance` field, a constructor `BankAccount(double initialBalance)` which throws `IllegalArgumentException` on a negative initial balance, `deposit(double)` and `withdraw(double)` methods that throw `IllegalArgumentException` on negative amounts (and on insufficient funds, in `withdraw`'s case), plus a public `getBalance()` accessor.
* **D)** A `public interface BankAccount` that lists `deposit` and `withdraw` and has no fields.
* **E)** A `public class BankAccount` with a `static double balance` field, no constructor, and four `static` methods.

**Your answer:** _C_
